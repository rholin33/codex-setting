# 提交信息生成器

每次执行时**必须**实时获取暂存区最新状态，不要使用缓存或假设。

## 实时获取信息

```bash
# 暂存文件列表
STAGED_FILES=!`git diff --cached --name-only`

# 变更统计
STATS=!`git diff --cached --stat`

# 具体变更内容（全部显示）
DIFF=!`git diff --cached`

# 当前分支
BRANCH=!`git branch --show-current`

# 最近提交记录（分析风格）
RECENT_COMMITS=!`git log --oneline -10`
```

## 忽略规则

避免扫描以下锁文件：

- pnpm-lock.yaml
- package-lock.json
- yarn.lock
- bun.lockb

## 严格的生成规则

### 1. 必须分析历史风格（强制执行）

在生成提交信息前，**必须**严格分析 `RECENT_COMMITS` 中最近 5-10 条记录的风格，包括：

- 提取 emoji 前缀风格（中文/英文）
- 确定主体内容格式
- 分析提交类型分布

**如果不遵循历史风格，命令执行失败！**

### 2. 项目风格匹配策略

根据项目历史提交记录分析，本项目有以下明确风格：

#### 类型与对应 emoji（严格匹配）

| 场景               | 必须使用的 emoji | 示例                              |

| 新增工具/组件      | ✨               | ✨ 新增 sidebar 自动化生成工具    |
| 配置优化、路径调整 | 🔧               | 🔧 优化图片资源路径并更新配置     |
| 资源添加、格式优化 | 📦               | 📦 新增文章图片资源并优化图片格式 |

| 修复 Bug           | 🐛               | 🐛 修复侧边栏显示问题             |

| 代码格式化         | 🎨               | 🎨 代码格式化优化                 |

#### 内容格式要求

1. **第一行（标题）**：
   - 必须包含 emoji 前缀
   - 中文描述，简短清晰（< 50 字）
   - 使用中文标点

2. **主体部分**：
   - 使用 `- ` 列表格式
   - 每个条目描述一个具体变更
   - 语言要与项目保持一致

### 3. 不允许的风格（严格禁止）

- 不允许使用英文 emoji 前缀（如 :sparkles:）
- 不允许使用英文标点（如 .）
- 不允许标题过长（超过 50 字）
- 不允许没有明确的变更描述

## 输出要求

**必须使用 markdown 代码块格式输出**，将提交信息包裹在 ```markdown 代码块中，不要直接输出渲染后的文本。

示例输出格式：

```markdown
✅ 新增 SelectColor 组件测试

- 添加渲染测试：验证 select 元素和 label 文本
- 添加用户交互测试：验证点击后显示 3 个选项
```

注意：

- 不要展示获取的 git 信息内容（STAGED_FILES、DIFF 等）
- 只输出最终的提交信息，且必须在代码块中
