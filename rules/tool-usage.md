# Codex 工具使用规则

## 核心原则

- 文件读取、搜索和编辑优先使用 Codex 提供的工具与高信号命令，保持操作可审计、可回滚。
- 任何规则都不能覆盖系统、开发者或当前用户指令；冲突时以上层指令为准。
- 遵守当前会话的 sandbox 和 approval policy；需要越权、联网或危险操作时按 Codex 提供的审批机制处理。

## 文件操作

- 搜索文件优先使用 `rg --files`，按名称过滤时使用 `rg --files | rg '<pattern>'`。
- 搜索内容优先使用 `rg`；如果 `rg` 不可用，再使用系统自带替代命令。
- 读取文件可使用 `sed -n`, `nl -ba`, `head`, `tail` 等只读命令；避免一次性输出超大文件。
- 手工编辑文件必须使用 `apply_patch`。
- 不使用 `cat > file`, heredoc、重定向或临时脚本来写文件，除非是工具生成的机械输出且明显更安全。
- 可用格式化器、代码生成器或包管理器执行项目已有的机械修改，但运行前确认其作用范围。

## Git 操作

- 允许只读命令：`git status`, `git diff`, `git log`, `git show`, `git blame`, `git branch`。
- 不主动执行 `git add`, `git commit`, `git push`, `git reset`, `git checkout` 等会改变仓库状态的命令，除非用户明确要求。
- 绝不使用 `git reset --hard` 或 `git checkout -- <path>` 回退用户改动，除非用户明确要求该具体操作。
- 遇到非本人改动时，先识别影响范围；相关改动要协同处理，无关改动直接忽略。

## GitHub 与网络

- 可用 `gh` 做只读查看：`gh pr view/list`, `gh issue view/list`, `gh repo view`, `gh run view`, `gh api`。
- `gh` 的创建、合并、关闭、删除等写操作必须有用户明确授权。
- 浏览、下载依赖或访问网络时，遵守当前环境权限；如果 sandbox 阻止关键网络操作，按 Codex 规则请求授权。

## 构建与测试

- 优先运行与改动最相关的测试、类型检查或 lint，再考虑更广的验证。
- 使用项目已有命令，例如 `npm test`, `npm run build`, `npx tsc --noEmit`, `npx vue-tsc --noEmit`, `cargo test`, `pytest`。
- 不为验证引入新的测试框架或格式化工具。
- 遇到无关失败时，不扩大修复范围；在最终说明中标出失败和关联性。

## 危险操作

- 删除、移动、大规模覆盖、权限修改、进程终止等操作必须谨慎；未被明确要求时不要执行。
- 不执行 `rm -rf`, `sudo`, 系统级安装、全局配置重写等高风险命令，除非用户明确要求并且目的清楚。
- 处理凭据、token、私钥、环境文件时只做必要读取和最小变更，不在回复中泄露敏感值。

## 执行前检查

- 这个操作是否会写文件、改仓库状态、联网或影响系统环境？
- 是否已有更直接、更局部的 Codex 工具或只读命令可完成？
- 是否会覆盖用户未要求修改的内容？
- 是否需要先告知用户当前正在做什么？
